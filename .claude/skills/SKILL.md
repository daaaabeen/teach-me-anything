# Teach Me Anything — 体系化学习工作流（总入口）

把「学会一个新主题」变成有标准、有验收、可复用的流程：开题定级（L1 能聊 / L2 能讲 / L3 能用）→ 知识地图 → 逐模块「讲解-苏格拉底追问-测验」→ 实践（L3）→ 费曼终验 + 速查笔记 + 分享 PPT。

<HARD-RULE>
本文件只做意图分发，不复制具体流程。执行任何学习活动前，必须完整读取仓库根目录的 `METHODOLOGY.md`——它是所有阶段做法、产出与完成判据的唯一来源。
</HARD-RULE>

## 意图路由

收到用户请求后，先判断意图，再加载对应子技能执行：

### 1. 学习意图（开题 / 续学）

触发：`/learn [主题]`、「学习 X」「我想学 X」「继续学」「接着上次学」

→ 读取并执行 `.claude/skills/learn/SKILL.md`。要点速览（细节以该文件为准）：

- **有参数且匹配 `topics/` 已有主题** → 读其 `state.md`，播报进度，从断点继续
- **有参数且是新主题** → 走方法论「① 开题」：摸底 → 定层级 → 控范围 → 建档
- **无参数** → 扫描 `topics/*/state.md`：0 个进行中引导开新主题；1 个直接续学；多个列表让用户选

### 2. 进度查询意图（只读）

触发：`/learn-status`、「学习进度」「我学到哪了」「学习情况」

→ 读取并执行 `.claude/skills/learn-status/SKILL.md`：扫描 `topics/*/state.md`，按「进行中 / 已结题」分组输出进度表（主题、层级、阶段、模块进度、最后学习日期），不修改任何文件、不进入学习流程。

### 3. 意图不明

用户只说「我想学点东西」之类 → 按学习意图处理，先问想学什么。

## 执行铁律（任何路径都不可妥协）

- **先展示后确认**：要用户确认的产物（地图、设计、范围等）必须先在独立消息里完整呈现（关键产物渲染到浏览器）再发确认型问题，绝不让用户对没看到的内容拍板
- **流程透明**：开题先讲清流程与产出；元流程术语（范围、slug、知识地图、苏格拉底、gate）首次出现即解释
- **测验 gate 不通过不进下一模块**；不通过不重考原题，换角度重讲后出新题
- **苏格拉底先问后讲**：每模块讲解后必有 2-3 个引导性追问，用户回应后才进测验
- **趁热沉淀**：模块过关立即写 `notes/`，立即更新 `state.md`（含错题记录）
- **状态先行**：每完成一个模块或阶段先更新 `state.md` 再继续，保证任意时刻中断可续学
- **实践环节只评审不代写**（L3 技术类）
- 结题 PPT 生成前确认用户是否需要，产出到 `topics/<slug>/share/slides.pptx`
- 关键文档产物（知识地图、summary）渲染成 HTML 在浏览器中展示供用户审阅
