# Teach Me Anything — 可复用学习工作流设计

日期：2026-06-11
状态：已与用户确认定稿

## 背景与目标

AI 发展节奏快，用户需要持续学习新知识。本项目在 `teach-me-anything` 目录建立一套可复用、体系化的「学习新知识」工作流：由 Claude 驱动的交互式流程，把任意新主题从开题带到验收，并沉淀可查阅的知识档案与可对外分享的 PPT。

**适用知识类型**：通用框架，按类型适配实践环节——技术/工程类、AI/ML 原理与论文、领域概念/软知识。

**明确不做**：间隔复习/抗遗忘排期（spaced repetition）。工作流只负责「学会」，长期查阅依靠沉淀的笔记档案。

## 总体形态

- 核心资产是 `METHODOLOGY.md`（方法论文档）：人可读、与工具无关，定义五阶段流程与各环节标准。
- Claude Code skill 是方法论的执行器：执行时引用方法论文档作为行为依据，修改方法论即改变行为，无需改 skill。
- 命令面刻意做小，只有两条命令。

## 目录结构

```
teach-me-anything/
├── CLAUDE.md                 # 工作区说明（保持极薄：工作区定位 + 指向 METHODOLOGY.md 与 skills，不重复方法论内容）
├── METHODOLOGY.md            # 方法论核心：五阶段定义、提问原则、测验与验收标准
├── .claude/skills/
│   ├── learn/SKILL.md        # 主命令 /learn：新主题开题，或自动断点续学
│   └── learn-status/SKILL.md # /learn-status：总览所有主题进度
└── topics/
    └── <主题slug>/           # 每个主题一个档案目录
        ├── state.md          # 进度状态（人可读 markdown）
        ├── map.md            # 知识地图（阶段②产物）
        ├── notes/            # 逐模块学习笔记（边学边沉淀）
        ├── exercises/        # 实践产物：代码、推导、案例分析（L3）
        ├── summary.md        # 结题速查笔记
        └── share/
            └── slides.pptx   # 对外分享 PPT
```

## 命令设计

| 命令 | 行为 |
|------|------|
| `/learn <主题>` | 匹配到已有主题 → 读 `state.md` 断点续学；新主题 → 走开题流程 |
| `/learn`（无参数） | 扫描 `topics/`，列出进行中主题让用户选择（仅一个时直接续学；没有进行中主题时引导开启新主题） |
| `/learn-status` | 列出所有主题：目标层级、当前阶段、最后学习日期 |

五个阶段是流程内部状态，不暴露为独立命令。

**主题 slug 约定**：开题时由 Claude 提议一个简短英文 slug（如「RAG 检索增强生成」→ `rag`），用户确认后作为 `topics/` 下的目录名；`state.md` 标题保留中文全名。

## 五阶段流程

### ① 开题（一次对话完成）

用户给出主题，Claude 执行三步：

1. **摸底**：2-3 个问题探明已有基础与学习动机（决定后续类比锚点与跳过的内容）
2. **定目标**：用户选定目标层级——**L1 能聊**（知道是什么、为什么重要）/ **L2 能讲**（费曼式讲清原理）/ **L3 能用**（做出东西或解决实际问题）
3. **控范围**：主题过大时主动建议拆小（如「学 Rust」→ 建议「Rust 所有权模型」），颗粒度以一两周可拿下为宜

产出：`state.md` 初稿。

### ② 建图

Claude 先做研究（官方文档、权威资料），输出知识地图：模块清单、依赖顺序、每模块的「学完能回答什么问题」。地图按目标层级裁剪——L1 砍实现细节，L3 加实践模块。用户确认或修改后定稿为 `map.md`。

### ③ 循环学习（核心循环，逐模块）

每个模块固定三步：

1. **讲解**：基于用户已有基础讲解，用其熟悉的概念做类比，不照搬教材
2. **苏格拉底追问**：不直接往下讲，抛 2-3 个引导性问题让用户推理（如「若 X 换成 Y 会怎样？」），暴露「以为懂了其实没懂」的缺口
3. **测验 gate**：3-5 题（概念辨析 + 应用题），通过才标记模块完成。**不通过不重考原题**——定位错因，针对薄弱点换角度重讲，再出新题

每模块过关时将要点沉淀到 `notes/`，趁热打铁。测验错题记入 `state.md` 测验记录（供结题 PPT 误区页取材）。

### ④ 实践（仅 L3）

按知识类型适配：

- **技术/工程类**：在 `exercises/` 写最小可运行项目，Claude 只评审不代写
- **AI/ML 原理类**：手推关键公式或实现玩具版本（如百行实现 attention）
- **软知识类**：取用户工作中的真实场景做案例分析

### ⑤ 结题（费曼终验 + 双产物）

角色反转：用户向 Claude 讲整个主题，Claude 扮演「爱追问的外行」挑漏洞。漏洞回炉补课，通过后产出：

1. **`summary.md`**：一页式自用速查笔记（核心模型、关键结论、易错点、延伸方向）
2. **`share/slides.pptx`**：对外分享 PPT（用 pptx 能力生成，默认生成、可跳过），结构按「教别人」设计：
   - 动机/背景（为什么值得关心）
   - 核心概念地图（取自 `map.md` 主干）
   - 原理讲解（复用费曼终验中讲得最顺的表述与类比）
   - 实战/案例（L3 主题直接用 `exercises/` 产物做演示素材）
   - 常见误区（取自用户测验错题——亲自踩过的坑大概率也是同事会踩的）
   - Q&A 预埋（预测同事可能的提问 + 参考答案，放备注或附录）

PPT 内容取材于学习过程中的真实数据（错题、卡点），这是相比凭空生成 PPT 的独特优势。

## 状态管理与断点续学

`state.md` 使用人可读 markdown（非 JSON）：用户可直接查看与手改，Claude 易解析。包含：

- 主题、目标层级、当前阶段、开题日期、最后学习日期
- 模块进度 checklist（未开始/学习中/已过关 + 过关日期）
- 测验记录（错因要点）
- 变更日志（如目标层级降级）

示例：

```markdown
# RAG 检索增强生成

- 目标层级: L2 能讲
- 当前阶段: ③ 循环学习
- 开题日期: 2026-06-11
- 最后学习: 2026-06-15

## 模块进度
- [x] 1. 为什么需要 RAG（已过关 6/12）
- [x] 2. Embedding 与向量检索（已过关 6/14）
- [ ] 3. 分块策略 ← 当前
- [ ] 4. 重排与评估

## 测验记录
- 模块2 第一次未过：混淆余弦相似度与点积的适用场景 → 已重讲过关

## 变更日志
- 6/14 目标从 L3 降为 L2（近期没有实践时间）
```

每完成一个模块/阶段即更新 `state.md`，session 随时中断不丢进度。

## 边界处理

- **中途降级目标**（L3 → L2）：随时允许，记入变更日志，地图相应裁剪
- **模块卡住三次以上**：Claude 主动建议拆出前置子主题先学
- **主题过大**：开题阶段即建议拆分，不带病开工

## 验证方式

搭建完成后用真实主题 dogfood：选一个用户真实想学的小主题（L1/L2 级别）完整跑通五阶段，中途故意中断一次 session 验证断点续学，检查产出笔记与 PPT 质量。跑通才算交付。

## 已考虑并否决的方案

- **极简单命令 + 方法论文档**（Claude 自由发挥）：执行质量不稳定，缺验收标准，断点续学弱
- **多代理流水线**（研究/出题/评审代理）：过度设计，学习本质是与导师的对话
- **内置间隔复习**：用户明确不需要，学完即止
