# 模块 4：失败模式与防护

学习日期：2026-06-12 ｜ 测验：4/4（苏格拉底环节有一个概念卡点，已纠正并换角度复测通过）

## 五大失败模式

| 模式 | 特征 | 病因 |
|------|------|------|
| 死循环 | 永远到不了终止条件 | 目标不可验证（「足够好」）或不可达 |
| 无进展空转 | **看起来很忙**但状态不前进；典型形态：修 A 挂 B、修 B 挂 A 的摆动 | 失败尝试没落盘、同路径重试 |
| 目标漂移 | 修登录 bug 修成重构权限模块，每步局部合理 | 目标没每轮重新锚定，被中间过程稀释 |
| 上下文溢出 | 压缩丢关键约束后行为失控 | 长任务塞爆窗口 |
| 成本失控 | 单 agent ≈ 4× 对话 token，多 agent ≈ 15× | 以上四种的共同结算方式 |

**AutoGPT（2023）案例**：无可验证目标 + 零护栏 → 死循环 + 烧钱是必然结果。惊艳与不可用同源。

## 防护四支柱（纵深防御，缺一不可）

| 支柱 | 防什么 | 关键性质 |
|------|--------|----------|
| 硬性迭代上限 | 死循环 | 最后保险，无条件停 |
| token 预算 | 成本失控 | 管单轮开销大的场景（迭代上限管不了） |
| 无进展检测 | 空转/摆动 | 唯一能**提前**止损的（空转会体面耗满全部轮次） |
| 明确终止条件 | 一切的前提 | 四个里唯一让循环**因成功而停**的 |

防目标漂移：目标与约束写入 state 文件，**每一轮**回读并注入 prompt 固定位置（不是每 session 一次——稀释每轮都在发生）。

## 本次踩坑记录（PPT 误区页素材）

- ❌ 初判「死循环 = 有进展但停不下来」→ ✅ 死循环的定义只是「到不了终止条件」，不保证有进展
- 真正的辨析维度是**可见性**：死循环在监控上显形（重复日志、轮数飙升）；**空转才更危险**——每轮都有新产出、日志健康，「看起来在干活」，只有对比多轮状态才能发现进度没动 → 发现晚、烧得多

## 元认知锚点

学习方法论里的对应物：「卡住三次拆前置子主题」= 无进展检测 + 升级；「不重考原题」防 A→B→A 摆动。
