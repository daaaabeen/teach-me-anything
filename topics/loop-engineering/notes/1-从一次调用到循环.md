# 模块 1：从一次调用到循环——agent 的本质

学习日期：2026-06-11 ｜ 测验：4/4 一次通过

## 核心模型

三种用法的光谱：

| | 单次调用 | 固定 workflow | agent loop |
|--|---------|--------------|-----------|
| 例子 | 网页版 ChatGPT 问答 | 总结→生成→测试 流水线 | Claude Code 修 bug |
| 模型角色 | 一次性文本函数 | 流水线填空工人 | 现场决策者 |
| 下一步由谁定 | 没有下一步 | 代码写死 | **模型看结果现场决定** |
| 错误的命运 | 交卷即终点 | 按预设规则重试 | 成为下一轮的输入 |

**agentic loop = 思考 → 行动（调工具）→ 观察（结果接回来）→ 再思考……直到自判完成。**

## 关键洞察

1. **能力来自循环放大**：同一个模型，循环把「一次性智力」放大成「持续工作能力」。失败输出（如测试报错）是定位问题的线索，不是终点。
2. **loop vs workflow 的分界刀**：分支是写死的还是模型现场决定的。CI 自动重试不是 loop——重试策略是预设的，运行时没有决策。
3. **循环 = 记忆 + 手 + 反馈自动接回来**。只有记忆（贴历史给单次调用）不够，模型还要能自己行动、自己观察。
4. **停下来问人不是循环失灵**：agent 判断需要循环外部的输入而主动让出控制权，是终止/暂停条件的一种，属于循环设计的一部分。

## 主线定位

本模块讲的是**内循环**（一条消息之内，harness 已实现）。「下一条消息谁来发」的**外循环**才是 loop engineering 的主战场——模块 2 起逐步展开。
