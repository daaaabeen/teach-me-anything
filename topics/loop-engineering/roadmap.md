# 进阶路线图：AI Agent 循环工程（入门 → 精通）

> 生成于 2026-06-15 ｜ 配套 summary.md 使用
> 本工作流把你带到了「L2 能讲」= **入门**。这份指明往「能设计、调优、运维生产级 loop」走的路。工作流不代为执行，自学按阶推进。

## 你现在在哪

- **已达成**：能讲清内/外循环之别、六大构件、评判者-执行者分离、五种失败模式与四支柱防护、prompt→context→loop 的演进脉络。
- **还没碰**：① 一手论文（你的演进史目前全建在二手博客上）；② 真实 harness 的源码级实现（终止判定、上下文压缩、状态恢复到底怎么写的）；③ 亲手做过一个带防护的 loop；④ 多 loop 编排、评测与可观测性。
- **差距本质**：你现在是「懂概念」，精通要的是「读过真代码 + 自己踩过坑 + 能权衡取舍」。

## 进阶阶梯

### 阶梯 1 ── 用一手材料换掉二手骨架（约 1 周，必做）
- **为什么先做**：你要讲给同事，演进史里的「谁哪年提出了什么」最容易在二手转述中走样；先把骨架钉在一手来源上。
- **学什么 + 资料**：
  - *ReAct: Synergizing Reasoning and Acting in Language Models*（Yao et al., 2022, arXiv:2210.03629）——内循环的起点，读它怎么把 reasoning 和 acting 交替起来。
  - *Reflexion: Language Agents with Verbal Reinforcement Learning*（Shinn et al., 2023, arXiv:2303.11366）——「自我反思」如何进循环。
  - Lilian Weng, *LLM Powered Autonomous Agents*（lilianweng.github.io, 2023）——最权威的 agent 综述，规划/记忆/工具三件套的体系化讲法。
  - Anthropic, *Building Effective Agents*（2024, anthropic.com 工程博客）——workflow vs agent 的官方界定，和你模块1的「刀」直接对照。
- **怎么算到位**：能指出你 map.md 里哪些二手表述与一手有出入，并修正。

### 阶梯 2 ── 读真实 harness 的源码（约 1-2 周）
- **为什么**：概念到工程的鸿沟全在实现细节里——终止条件、无进展检测、上下文压缩、断点恢复，博客都一笔带过，代码里才有真相。
- **学什么 + 资料**：
  - 读一个开源 agent 循环实现（如 LangGraph 的 graph/state 机制官方文档与示例、或任一开源 coding agent 的主循环），重点看：迭代上限写在哪、状态怎么持久化、压缩触发条件。
  - **零成本实践**：在 Claude Code 里真实跑一个多步任务，开 verbose，肉眼观察「思考→工具→观察→再决定」和它何时终止——把抽象的内循环看成具体行为。
- **怎么算到位**：能画出某个真实 harness 的外循环状态机（含失败分支），而不只是教科书里的五步圈。

### 阶梯 3 ── 亲手做一个带防护的最小 loop（约 1-2 周，分水岭）
- **为什么**：精通的标志是「做得出 + 知道为什么这么权衡」，不是「讲得出」。这一步从 L2 跨到 L3。
- **学什么**：用任意框架或裸 API，实现 planner→executor→verifier 的最小循环，**独立的 verifier**（应用你学的评判者-执行者分离），装齐四支柱：迭代上限、token 预算、无进展检测、明确终止。然后故意喂它会触发死循环/空转/目标漂移的任务，看防护是否真拦得住。
- **怎么算到位**：四种失败模式你都能人为复现、并被你的防护捕获；能讲出每道防护的误杀/漏杀权衡。

### 阶梯 4 ── 编排、评测与运维（持续）
- **为什么**：单 loop 之上是你 summary 里点名的「下一个瓶颈」——多 loop 协作；生产化还要可观测性与成本治理。
- **学什么 + 资料**：
  - 多 agent / 多 loop 编排模式（Anthropic 多 agent 研究、LangGraph 的 multi-agent 文档）。
  - Agent 评测：怎么给一个会自主决策的系统打分（轨迹评测、任务成功率），而不只看最终输出。
  - 可观测性与成本：trace、token 预算熔断在真实系统里怎么落地。
- **怎么算到位**：能为一个真实重复性工作（如值班修 bug）设计出可上线的 loop 方案——含触发、隔离、验证、熔断、回退给人。

## 下一个主题建议（可直接 `/learn`）

- **`context-engineering`（context engineering 上下文工程）**——loop 的「楼下」，压缩/检索/记忆是 loop 稳定的地基，和本主题直接接壤。
- **`agent-evaluation`（Agent 评测与可观测性）**——阶梯 3-4 的硬骨头，单独成题更扎实。
- **`multi-agent-orchestration`（多 agent 编排）**——你 summary 标记的「疑似下一个瓶颈」，待入门铺垫够了再开。
